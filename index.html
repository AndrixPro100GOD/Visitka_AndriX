Обо мне

Сжатая выжимка; развёрнутое резюме — на hh.ru.

Full-Stack .NET разработчик с более чем 3 годами коммерческого опыта. Специализируюсь на сложных backend-системах, real-time сценариях и миграции legacy на современный стек без потери бизнес-логики.

За последний год с нуля разработал и вывел в коммерческую эксплуатацию TMS — систему управления грузовыми контейнерами и логистикой: полный цикл заявок, обновления в реальном времени (SignalR и Redis), интерактивная схема терминала, миграция старой системы на .NET 8, контейнеризация и поставка клиентам через Docker.

Ранее более двух лет занимался Unity: коммерческие казуальные игры с нуля и свыше 200 часов занятий по C# и Unity — более 10 учеников выпустили первые полноценные проекты.

Сильные стороны: ASP.NET Core, EF Core, PostgreSQL, оптимизация производительности, чистый код. Уверенно работаю с Docker, Git, REST и real-time. Ищу роль Full-Stack / Backend .NET в продуктовой команде.

TMS — Transport Management System

Система управления грузовыми контейнерами и логистикой: от заявки до завершения, с учётом стоков, ЖД-платформ и коммерческой поставки клиентам.

  • Разработка TMS с нуля на C# .NET 8, ASP.NET Core и Blazor (MudBlazor): полный end-to-end цикл обработки заявок.
  • Real-time статусы контейнеров и заявок: gRPC, SignalR и Redis — изменения видны всем пользователям без перезагрузки.
  • Интерактивная схема терминала (стоки, ЖД-платформы): собственный fork Z.Blazor.Diagrams и Canvas API.
  • Миграция legacy на .NET 8 по ТЗ: библиотеки по сущностям, автоматическая синхронизация данных, переход без потери информации.
  • Вывод в коммерцию: разделение сборок (внутренняя / клиентская), Docker-образы для внешних компаний, успешные продажи продукта.
  • Практики поставки: Git и Docker, GitLab CI, нагрузочное тестирование K6, интеграционные тесты, в т.ч. связка Android-приложения с Backend API.
  • Оптимизация EF Core и PostgreSQL, развитие REST API: быстрее обработка заявок, меньше ошибок в данных.
